# STS2型ゲームの観察マップ

## 目的

Slay the Spire 2 のような完成度の高いデッキ構築ローグライクを、初心者が「何を見ればよいか」に分解する。

この資料は、既存作品の内容を写すためのものではない。プレイしながら設計を観察し、自作ゲームへ変換するためのチェックリストである。

## 1. ラン全体

観察すること:

- 1ランがどのくらいの時間で終わるか
- 序盤、中盤、終盤で考えることがどう変わるか
- 負けたときに「次はこうしよう」と思えるか
- ランダム要素が、理不尽ではなく選択の変化になっているか

自作ゲームへ変換する:

- 縦切り版は1Actだけにする
- 1Actは10から15ノードで終わるようにする
- 最初の敵は学習用、エリートはビルド確認、ボスは総合試験にする

## 2. カード

観察すること:

- スターターカードはどれくらい単純か
- コモンカードはどうデッキの土台になるか
- アンコモンはどんな条件やシナジーを持つか
- レアはどれくらいプレイ方針を変えるか
- 弱いカードにも、特定状況で欲しくなる理由があるか

自作ゲームへ変換する:

| 役割 | 自作で用意する例 |
| --- | --- |
| 基準攻撃 | 1コスト、6から8ダメージ |
| 基準防御 | 1コスト、5から8ブロック |
| ドロー | 直接火力を持たない代わりに選択肢を増やす |
| リソース生成 | 次ターンや専用リソースと交換する |
| ビルド軸 | 特定のタグや状態を伸ばす |

## 3. 敵

観察すること:

- 最初の敵が何を教えているか
- 複数敵の戦闘で、全体攻撃の価値が出ているか
- エリートがどの弱点を突いてくるか
- ボスのギミックが、それまでの道中で予告されているか

自作ゲームへ変換する:

| 敵タイプ | 役割 |
| --- | --- |
| 小型単体 | 攻撃と防御の基本を教える |
| 小型複数 | 全体攻撃、優先対象、被ダメージ管理を教える |
| 防御型 | スケーリングや毒のような別解を促す |
| 高火力型 | 敵意図を見て防御する重要性を教える |
| エリート | デッキの偏りを罰する |
| ボス | Act内の学習内容をまとめる |

## 4. 遺物

観察すること:

- 遺物が単なる数値上昇だけでなく、ビルドの方向を変えているか
- いつ発動するかが分かりやすいか
- ボス報酬は強い代わりにデメリットや制約があるか

自作ゲームへ変換する:

| 遺物タイプ | 例 |
| --- | --- |
| 開幕効果 | 戦闘開始時にリソースを得る |
| 条件効果 | N枚目のカード、初回攻撃などで発動 |
| 報酬補助 | Gold、カード報酬、ショップに影響 |
| リスク付き強化 | Energy増加とデメリットをセットにする |

## 5. イベント

観察すること:

- HP、Gold、カード、遺物のどれを交換しているか
- 選択肢が「明らかな正解」になっていないか
- ランの状況によって選びたいものが変わるか

自作ゲームへ変換する:

```text
イベントの基本形:
  選択肢A: 今すぐ得をするが、長期的なデメリット
  選択肢B: 今は痛いが、長期的なメリット
  選択肢C: 何もしない、または小さな安全策
```

## 6. UI

観察すること:

- 敵の次行動がどれくらい明確に見えるか
- カードを使える、使えない理由がすぐ分かるか
- 山札、捨て札、廃棄札を確認しやすいか
- 報酬選択で、今のデッキと比較しやすいか
- 初見で最低限の操作が分かるか

自作ゲームへ変換する:

- 戦闘画面では、敵意図、HP、Block、Energy、手札を最優先で見せる
- 説明文は短くし、キーワードはツールチップで補足する
- 重要な状態異常はアイコンだけでなく数値も表示する

## 7. バランス

観察すること:

- 序盤の攻撃カードがどれくらい価値を持つか
- 防御を軽視するとどのあたりで苦しくなるか
- カード削除やアップグレードの価値が高いか
- どの報酬を選ばないことが正解になる場面があるか

自作ゲームへ変換する:

- まず基準値を作る
- そこからカードごとに条件、デメリット、シナジーで上下させる
- 10ラン未満の感想で大きく変えすぎない
