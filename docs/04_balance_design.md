# バランス設計

## 目的

感覚だけで調整せず、プレイログと設計目標を使って数値を決める。

この段階の数値は正解ではない。比較できる仮説として置く。

## 初期目標値

Act 1 縦切り版の目標。

| 指標 | 目標 |
| --- | ---: |
| 初見勝率 | 20%から35% |
| 慣れたプレイヤーの勝率 | 55%から70% |
| 通常戦闘の平均ターン | 3から5 |
| エリート戦闘の平均ターン | 5から8 |
| ボス戦闘の平均ターン | 8から12 |
| Act 1終了時の平均HP | 最大HPの35%から60% |

## カード数値の初期基準

Energy 1あたりの基準値。

| 種別 | 基準 |
| --- | ---: |
| 単体ダメージ | 6 |
| 全体ダメージ | 4 |
| Block | 5 |
| 1ドロー | 2から3ダメージ相当 |
| 状態異常1付与 | 3から5ダメージ相当 |
| Energy獲得 | 条件付きにする |

例:

- 1コスト攻撃: 6ダメージ
- 1コスト防御: 5ブロック
- 2コスト攻撃: 13から15ダメージ
- 0コストカード: 直接火力を低くし、条件やデメリットを付ける

## レアリティ方針

| レアリティ | 役割 |
| --- | --- |
| starter | 弱いが分かりやすい |
| common | デッキの骨格 |
| uncommon | 方針を作る |
| rare | ビルドを変える |
| special | イベント、敵、特殊報酬 |

Rareは単純に強いだけではなく、「取るとプレイ方針が変わる」ものにする。

## 敵設計

通常敵:

- 1つの学習テーマだけ持たせる
- 例: 防御の重要性、全体攻撃の価値、状態異常の危険

エリート:

- デッキの弱点を強く問う
- 報酬が大きいため、避けるか挑むか悩ませる

ボス:

- Act全体の学習内容をまとめて試す
- 長期戦になるため、成長不足のデッキを倒す

## 調整ログで見る指標

毎ラン記録する。

- ランID
- 使用キャラ
- 到達階層
- 死因
- 取得カード
- 削除カード
- 取得遺物
- 各戦闘のターン数
- 各戦闘の被ダメージ
- ボス撃破可否

カードごとに見る。

- 提示回数
- 選択回数
- 使用回数
- 勝利ランでの採用率
- 敗北ランでの採用率
- アップグレード率

## 調整判断

| 現象 | 可能性 | 対応 |
| --- | --- | --- |
| どのランでも同じカードを取る | そのカードが強すぎる、または他が弱い | 強カードを下げる前に代替カードを強くする |
| 戦闘が長すぎる | 火力不足、敵HP過多 | 敵HPを下げるか、火力カードの質を上げる |
| 防御カードが選ばれない | 敵火力が低い、攻撃が万能 | 敵の予告火力を上げる |
| エリートを誰も踏まない | 報酬不足、難度過多 | 遺物価値を上げるか敵を弱める |
| ボスだけ極端に難しい | Act内の学習不足 | ボスギミックを通常敵で事前に教える |

## 重要な考え方

ゲームバランスは「全カードを同じ強さにする」ことではない。

目指すのは、状況によって評価が変わること。あるカードが弱いのではなく、「このデッキなら欲しい」と言える場面を作る。
