# 既存作品を参考にする範囲

## 結論

Slay the Spire 2 そのもののカード名、効果文、敵名、イベント文章、数値、画像、音声、内部ファイル内容をこのリポジトリへ複製することはしない。

代わりに、次を学習対象にする。

- 情報をどう分けて持つか
- プレイヤーに何を選ばせているか
- 1ランの流れをどう作っているか
- カード、敵、遺物、イベントがどう役割分担しているか
- プレイテストで何を見ればよいか

## OKな参考の仕方

| 観察対象 | 学ぶこと | この資料での置き換え |
| --- | --- | --- |
| カードの種類 | 攻撃、防御、継続効果、状態異常の役割 | `attack`, `skill`, `gear`, `status`, `curse` |
| 戦闘の流れ | ターン開始、手札、エネルギー、敵意図 | `docs/02_game_design_spec.md` |
| 遺物 | 条件付きの常時効果、ビルド補助 | `data/relics/sample_relics.json` |
| マップ | リスクと報酬のルート選択 | Monster, Elite, Rest, Shop, Event |
| 報酬 | 3択、スキップ、希少度 | カード報酬と遺物報酬 |
| ログ | バランス調整の材料 | `docs/04_balance_design.md` |

## NGな参考の仕方

- 既存作品のカードを同名・同効果・同数値で写す
- 敵やボスの名前、行動表、数値をそのまま写す
- イベント本文や選択肢を写す
- ゲーム内画像や音声を教材データとして保存する
- 内部ファイルを抽出して自作ゲームのデータに変換する

## 学習用の正しい変換

既存作品の1枚のカードを見たら、次のように抽象化する。

```text
元の観察:
  低コストの攻撃カード。序盤では強く、終盤は補助的。

抽象化:
  1コスト攻撃カードは、スターターより少し強く、複雑な条件を持たない。

自作ゲームへの変換:
  Bolt Throw
  Cost 1
  Deal 8 damage
```

敵も同じように抽象化する。

```text
元の観察:
  序盤に出る敵。弱い攻撃と予告行動で防御を教える。

抽象化:
  Act 1の最初の敵は、敵意図とブロックの重要性を教える。

自作ゲームへの変換:
  Rust Sentry
  Low HP
  Alternates between attack and defend
```

## なぜこの境界が大事か

既存作品をそのまま写すと、学習には見えても、実際には「なぜその数値なのか」「なぜその順番で登場するのか」を理解しにくい。

一度抽象化して、自分のゲーム用に置き換えると、設計意図を自分の言葉で説明できるようになる。これは将来、オリジナルゲームとして完成させるために重要。
